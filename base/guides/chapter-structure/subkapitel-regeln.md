# Unterkapitel-Regeln

## Mindestanforderungen

- Mindestens **1 Textseite** pro Unterkapitel
- Mindestens **2 Absätze** pro Unterkapitel
- Maximal **3 Gliederungsebenen** (z.B. 2.1.1, nicht tiefer)

## Keine verwaisten Unterkapitel

Wenn ein Kapitel unterteilt wird, müssen mindestens ZWEI Unterkapitel entstehen.

```
FALSCH:
2. Theoretische Grundlagen
   2.1 Low-Code-Plattformen
3. Analyse

RICHTIG:
2. Theoretische Grundlagen
   2.1 Low-Code-Plattformen
   2.2 Künstliche Intelligenz
3. Analyse
```

## Gliederungstiefe

| Ebene | Format | Beispiel |
|-------|--------|----------|
| 1 | X. | 2. Theoretische Grundlagen |
| 2 | X.X | 2.1 Low-Code-Plattformen |
| 3 | X.X.X | 2.1.1 Definition und Abgrenzung |

Dritte Ebene nur verwenden, wenn ein Unterkapitel sonst zu lang wäre (mehr als 4-5 Seiten).

## Überschriften

### Anforderungen
- Inhaltsbezogen und aussagekräftig
- Dezimalklassifikation
- Keine Fragen als Überschriften
- Keine Abkürzungen in Überschriften

### Beispiele

```
SCHLECHT:
3.1 Erste Herausforderung
3.2 Zweite Herausforderung

GUT:
3.1 Change Management bei der Einführung von KI
3.2 Strukturelle Voraussetzungen für Standardisierung
```

## Ausgewogenheit

Unterkapitel innerhalb eines Hauptkapitels sollten ähnlich lang sein.

```
PROBLEMATISCH:
3.1 (8 Seiten)
3.2 (2 Seiten)
3.3 (1 Seite)

BESSER:
3.1 (4 Seiten)
3.2 (4 Seiten)
3.3 (3 Seiten)
```

Wenn ein Unterkapitel deutlich kürzer ist: prüfen ob es mit einem anderen zusammengelegt werden kann.

## Checkliste

- [ ] Jedes Unterkapitel hat mindestens 1 Seite?
- [ ] Keine verwaisten Unterkapitel (immer mindestens 2)?
- [ ] Maximal 3 Gliederungsebenen?
- [ ] Überschriften sind inhaltsbezogen?
- [ ] Unterkapitel sind ähnlich lang?
